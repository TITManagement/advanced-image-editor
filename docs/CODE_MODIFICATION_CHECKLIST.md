# コード修正チェックリスト

このファイルは、コード修正時に設計指針を思い出すためのクイックリファレンスです。

## 🚨 修正前の必須チェック

```
[ ] 「修正前は動いていた」情報を確認
[ ] 既存コードの動作を理解
[ ] 最もシンプルな解決策を検討
[ ] 影響範囲を最小化できるか確認
```

## 💡 KISS原則の確認

```
[ ] 複雑な実装になっていないか？
[ ] ユーザーが「複雑すぎる」と感じないか？
[ ] オーバーエンジニアリングになっていないか？
[ ] もっとシンプルな方法はないか？
```

## 🎯 プラグイン特性の確認

### basic_plugin / density_plugin
```
[ ] リアルタイム処理が基本
[ ] スライダー変更→即座に画像更新
[ ] チャタリング対策が有効
```

### filters_plugin
```
[ ] 既存の画像処理パイプラインを使用
[ ] apply_all_adjustments経由で処理
[ ] 独自の処理フローを理解
```

## 🔄 修正後の検証

```
[ ] 元の機能が正常に動作するか？
[ ] 新しい問題を生んでいないか？
[ ] ユーザーの要求を満たしているか？
```

---

**重要**: このチェックリストは[設計指針](DESIGN_PRINCIPLES.md)の要約版です。
詳細な説明が必要な場合は、必ず設計指針ドキュメントを参照してください。
